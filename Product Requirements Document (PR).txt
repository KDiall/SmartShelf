# Product Requirements Document (PRD)

# SmartShelf

## The Operating System for Community Pharmacies

**Version:** Hackathon MVP v1.0

**Product Type:** Progressive Web Application (PWA)

**Target Market:** Community Pharmacies in Sierra Leone

**Prepared By:** SmartShelf Team

---

# 1. Executive Summary

SmartShelf is an offline-first pharmacy operating system that helps community pharmacies prevent medicine stockouts, reduce expiry-related losses, and simplify restocking operations.

The platform enables pharmacists to quickly record medicine sales through a touch-friendly interface, automatically track inventory levels, identify inventory risks, and generate supplier-ready restock requests.

The Hackathon MVP focuses on solving one critical problem:

> Pharmacies discover inventory problems too late.

SmartShelf helps them identify these problems before they impact patients or business operations.

---

# 2. Problem Statement

Many community pharmacies in Sierra Leone still rely on manual inventory tracking methods.

These methods create several challenges:

* Medicine shortages are discovered only after stock runs out.
* Patients cannot access needed medicines when inventory is unavailable.
* Pharmacy owners lose revenue due to stockouts.
* Medicines expire unnoticed on shelves.
* Restocking is manual, inconsistent, and reactive.
* Existing inventory systems are often too complex or require stable internet connectivity.

As a result, pharmacies struggle to maintain optimal inventory levels while continuing to serve customers efficiently.

# 7. Functional Requirements

## Module 1: One-Tap Sales Logging

### Description

Allows pharmacists to record medicine sales instantly.

### Requirements

* Display medicine tiles on dashboard.
* Allow one-tap sales logging.
* Deduct inventory automatically.
* Update inventory instantly.
* Work offline.

### Acceptance Criteria

* User can record a sale in under 2 seconds.
* Inventory updates immediately.
* No internet connection required.

---

## Module 2: Inventory Management

### Description

Tracks inventory levels locally.

### Requirements

* Store inventory locally.
* Display current stock quantities.
* Update quantities after each sale.
* Synchronize data when internet becomes available.

### Acceptance Criteria

* Inventory remains available offline.
* Inventory persists after browser refresh.
* Data synchronizes successfully when online.

---

## Module 3: Inventory Risk Intelligence

### Description

Identifies inventory risks before they impact operations.

### Stockout Risk Detection

Requirements:

* Calculate average medicine consumption.
* Estimate days remaining.
* Generate low-stock alerts.

Acceptance Criteria:

* System displays projected stockout dates.
* Alert appears before inventory reaches zero.

Example:

> Coartem projected to run out in 4 days.

---

### Expiry Risk Detection

Requirements:

* Store expiry dates.
* Monitor expiry timelines.
* Identify medicines approaching expiration.
* Display risk notifications.

Acceptance Criteria:

* Expiring medicines appear in alerts.
* User can view days until expiry.

Example:

> 15 packs of Amoxicillin may expire within 30 days.

---

## Module 4: Automated Restock Assistant

### Description

Generates reorder requests when inventory reaches critical levels.

### Requirements

* Detect low-stock thresholds.
* Recommend reorder quantities.
* Generate WhatsApp-compatible order message.

### Acceptance Criteria

* User can generate reorder request with one click.
* WhatsApp opens with pre-filled message.

---

## Module 5: Pharmacy Insights Dashboard

### Description

Provides a summary of inventory health.

### Requirements

Display:

* Total medicines
* Low-stock medicines
* Expiry-risk medicines
* Fast-moving medicines
* Slow-moving medicines

### Acceptance Criteria

* Dashboard loads in under 3 seconds.
* Metrics update automatically after sales.
