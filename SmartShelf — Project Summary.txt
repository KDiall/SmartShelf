SmartShelf — Project Summary
SmartShelf is an intelligent pharmacy inventory management system built for pharmacies in Sierra Leone. It runs as a Next.js 16 web application with a PostgreSQL database (hosted on Neon), uses WhatsApp for authentication and chatbot communication, and works offline via service worker caching and IndexedDB.
Tech Stack
- Frontend: Next.js 16 (App Router), React 19, TypeScript, Tailwind CSS v4, shadcn/ui, Zustand (state management), Lucide React (icons), Recharts (charts), react-select (searchable selects)
- Backend: Next.js API Routes (serverless), Prisma ORM, PostgreSQL (Neon), JWT authentication (Jose library), bcryptjs
- File & Offline Storage: UploadThing (file uploads for PDFs and images), Dexie.js (IndexedDB for offline data), Service Worker (runtime caching)
- WhatsApp Integration: Separate Node.js server using whatsapp-web.js, Express, deployed on Render — bridges a pharmacy phone number to the app via webhooks
- AI / Vector Search: OpenAI GPT-4o-mini (chatbot responses), Geneline (vector embeddings and search for RAG on uploaded treatment guidelines)
- Deployment: Vercel (main app), Render (WhatsApp server)
Core Architecture
The app is split into two parts: (1) the main Next.js application that handles the web dashboard, API routes, database, and AI chatbot, and (2) a separate WhatsApp bridge server (Node.js + whatsapp-web.js) that connects a pharmacy phone number to WhatsApp Web and forwards messages to the main app via webhooks.
How It Works — End to End
 1. Authentication: A user enters their phone number on the login page. The app sends a one-time password (OTP) to that number via WhatsApp. The user enters the OTP on the verify page, the app validates it against the database, and issues a JWT token stored in localStorage. Subsequent visits auto-login from the stored token. For demo/local development, phones 7000–7004 bypass WhatsApp and use the fixed OTP 123456.
 2. Role System: Three roles exist. Super Admin (phone 7000) has full access — can create pharmacies, manage users, upload medical guidelines, and reconnect the WhatsApp server. Admin users manage their assigned pharmacy's staff, medicines, and settings, and see "Pharmacy Information" on their settings page. Pharmacists can record sales, view stock, send restock orders, and edit their own profile (Full Name, Address, Location) — they cannot upload avatars or access admin pages.
 3. Dashboard: The home screen shows a hero card with the pharmacy name and date, a HealthRing visualization (a circular SVG gauge that displays the inventory health score as a percentage — green for good, amber for warning, red for critical), and risk summary cards below it (critical alerts, warnings, expired items). The health score is calculated by the risk engine: it starts at 100 and deducts 20 per critical alert (stock at or below zero) and 8 per warning (below reorder threshold but above zero).
 4. Inventory Management: The medicines page lists all stock items with name, unit, current quantity, reorder threshold, reorder quantity, and a colored stock status badge (OK = green, Low = yellow, Critical = red). Each medicine has a progress bar showing stock level relative to double the threshold. Users can add, edit, and delete medicines. Stock is tracked per pharmacy.
 5. Sales: The bulk sale page lets users record multiple sales at once using a dynamic line-item form. Each item has a searchable medicine selector (react-select) and quantity stepper. The app validates stock availability before recording and updates inventory in real time.
 6. Restock Orders: When stock runs low, users can generate a WhatsApp order message listing all items that need restocking. The app constructs a formatted message and opens WhatsApp with the supplier's number pre-filled.
 7. Reports: The reports page shows period-based summaries (Today, This Week, This Month, Custom Date Range) with total sales, transaction count, top-selling medicines, sales chart (bar chart), and low-stock alerts. Summary cards have colored left borders (teal for sales, blue for top seller, purple for transactions, red/teal for alerts).
 8. Medical Guidelines & AI Chatbot: Super admins and admins can upload Standard Treatment Guideline (STG) PDFs via the guidelines page. The PDF is stored via UploadThing, then sent to an AI ingestion pipeline. The chatbot (accessible via WhatsApp) uses RAG to search these uploaded guidelines and live inventory data before answering. For example, a pharmacist can message "Do we have paracetamol?" and get a reply with current stock, or ask "What is the treatment for malaria in children?" and get an answer grounded in the official uploaded protocol.
 9. WhatsApp Integration: The separate WhatsApp server runs whatsapp-web.js and maintains a persistent connection to WhatsApp Web. It watches for incoming messages, forwards them to the main app's webhook endpoint, and relays responses back to the user. The super admin can check connection status and trigger reconnection from the Settings page.
10. Offline Support: A service worker caches static assets and API responses, and an IndexedDB layer (Dexie.js) stores medicines and sales data locally. If the internet drops, the app still loads and displays cached data — staff can continue working until connectivity returns.
